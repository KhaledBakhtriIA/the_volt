## 1. Executive Summary

The Volt System is a sophisticated algorithmic trading infrastructure designed to ingest multi-source financial data, rigorously validate its integrity, engineer deterministic features, and train/serve machine learning models for market prediction.

Historically, the project relied on a monolithic, 600+ cell Jupyter Notebook (`AutoData_Analyst_v1_aymen.ipynb`) for feature engineering and model training. As of April 2026, the system has successfully completed a massive refactoring effort to liberate the core "Brain" of the platform from the research notebook into a highly-tested, decoupled, and production-ready Python codebase.

The system now features a **Live Trading Governance** layer including Alpha Validation, Portfolio Risk Management, and Algorithmic Execution Strategies, supported by **176 passing unit tests**.

## 2. Core Architectural Components

### A. Data Ingestion Layer (`data_api/`)
This is the front-line of the system, responsible for securely fetching, standardizing, and aggregating raw data.
* **10 Diverse Collectors:** Includes `market_collector` (OHLCV), `news_collector` (articles), `reddit_collector` (social sentiment), and `macro_collector` (FRED).
* **Fault Tolerance:** Built with strict isolation. If an API provider goes down or credentials expire, the specific collector returns an empty frame payload and the pipeline gracefully falls back to available data without crashing.
* **FastAPI Backend:** Orchestrates synchronous batch-collection jobs (`/collect/full`) and provides health/status checks for the cluster.

### B. The Production Pipeline (`src/canonical/`)
The canonical package represents the hardened production workflows of the system.
* **`FeatureStoreEngine`**: A rigorous statistical and financial-domain quality gate. It verifies column presence, asserts strict pricing domains (e.g. positive prices only), catches market-hour gaps, flags high Z-Score outliers, and drops invalid sequences before appending historic data into durable Parquet storage.
* **`FeatureEngineer`**: The newly extracted feature generation payload originally locked in the research notebook. It calculates over 150 standard indicators (Momentum, Volatility, Volume profiles) across vectorized NumPy arrays synchronously.
* **`XGBOptunaBundle`**: A self-contained, heavily optimized model training pipeline providing XGBoost classification, automatic OOF (Out-Of-Fold) stacking, and Optuna hyperparameter sweeps (budget-aware).
* **Real-time Scaffolding**: Components like `RealtimeRuntimeService` and `RedisFeatureCache` provide low-latency pipelines that consume live ticks, mutate an active rolling-window, and score ticks through model artifacts iteratively.

### C. The Core Domain Logic (`src/brain/`)
The brain module acts as a strict sandbox for pure math, indicator construction, and proprietary feature computation without side effects.
* **`trading_math.py`**: [NEW] Implements core financial math: Kelly Criterion for sizing, Expectancy Edge for alpha validation, and Drawdown tracking.
* **`features.py`**: Contains robust functional utilities like `calculate_std`, `calculate_atr`, and complex label generation structures (`add_3sigma_target`).
* Adheres to a strict functional design without reliance on external environment configs, making it heavily testable and optimized.

### D. Live Trading Governance (`src/canonical/`)
The newly implemented governance layer bridge signals to risk capital.
* **`risk_management.py`**: [NEW] Implements `PortfolioRiskModel` which enforces global drawdown kill-switches and dynamic position sizing using Kelly fractions.
* **`execution_strategy.py`**: [NEW] Provides algorithmic execution patterns, including **TWAP** (Time-Weighted Average Price) to minimize market impact by slicing orders.
* **`execution_gateway.py`**: [UPGRADED] Now fully integrated with the Risk Model to strictly size and validate every outgoing order contract.

## 3. Operations & Reliability Design

Volt is structurally designed around idempotency, auditability, and safety interlocks:
1. **Orchestrator Checkpoints:** Batch workflows generate `run_state` dictionaries mapping every execution step. If a complex job fails halfway, it resumes identically from the last successful checkpoint instead of rebuilding data from zero.
2. **Human-in-the-Loop Approval:** A `REQUIRE_HUMAN_APPROVAL` deployment gate exists within the `ModelRegistry`. New models generated by scheduled continuous learning loops are automatically placed in a `PENDING` state and prevented from active tick-scoring until manually promoted.
3. **Drift Detection & Loop Triggering:** Drift estimators automatically run Kolmogorov-Smirnov and PSI checks against live feature data. If feature deviation exceeds thresholds, an asynchronous learning loop invokes the Optuna Trainer to draft a replacement classifier.
